# AI Policy for Contributions

At Watermelon UI, we welcome contributions that leverage Artificial Intelligence (AI) tools, such as GitHub Copilot, ChatGPT, Claude, and others. We recognize that these tools can significantly boost productivity, improve code quality, and help developers learn.

However, we also require our contributors to use these tools responsibly to ensure the integrity, security, and quality of our codebase.

## Guidelines for AI-Assisted Contributions

1. **Human Review is Mandatory:** Never blindly copy-paste code generated by AI. You must thoroughly review, understand, and test any AI-generated code before submitting it in a Pull Request.

2. **Take Responsibility:** As the author of the Pull Request, you are fully responsible for the code you submit, regardless of whether it was written entirely by you, co-written with AI, or generated entirely by AI. You must be able to explain how the code works and why it was implemented that way.

3. **Check for Accuracy and Security:** AI models can hallucinate functionality, use deprecated APIs, or introduce subtle security vulnerabilities. meticulously verify that the AI-generated code uses correct APIs, follows security best practices, and correctly implements the intended logic.

4. **Follow Project Standards:** AI tools may not know our specific coding conventions, project structure, or architectural patterns. You must adapt AI-generated code to fit seamlessly into the Watermelon UI codebase, ensuring it passes our linting and formatting rules.

5. **Respect Copyright and Licensing:** Do not use AI tools to generate code that you know or suspect infringes on third-party copyrights or violates licensing terms. If you are unsure about the origin of a significant piece of AI-generated architecture or algorithm, try to find the original source or consider an alternative approach.

## Disclosure

We do not require you to disclose every minor instance of AI assistance (e.g., autocompletions or small refactors). However, if a significant portion of a new feature or complex component was generated by an AI, we encourage you to mention it in the Pull Request description (e.g., "Used Claude 3.5 Sonnet to help draft the initial structure for this component"). This helps maintainers understand the context and focus their review appropriately.

## Acceptance

By submitting a Pull Request to Watermelon UI, you agree to abide by this AI Policy. We reserve the right to reject contributions that we believe violate this policy or introduce poor-quality or insecure code, regardless of how it was generated.
