# Security Policy

## Supported Versions

We currently provide security updates for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| v1.0.x  | :white_check_mark: |
| < v1.0  | :x:                |

## Reporting a Vulnerability

We take the security of our software and the trust of our community very seriously. If you believe you have found a security vulnerability in Watermelon UI, please report it to us as described below.

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please send an email to **security@watermelon.sh** or appropriately contact the organization members privately if that address is unavailable.

### What to include

Please include the following information in your report:
- The type of issue (e.g., buffer overflow, SQL injection, cross-site scripting, etc.)
- Full paths of source file(s) related to the manifestation of the issue
- The location of the affected source code (tag/branch/commit or direct URL)
- Any special configuration required to reproduce the issue
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the issue, including how an attacker might exploit the issue

### How we will respond

You should receive a confirmation of receipt from us within 48 hours. We will then triage the issue to determine if it is a valid security vulnerability.

If the issue is confirmed:
1. We will evaluate the potential impact and assign a severity level.
2. We will work on a patch or mitigation to resolve the vulnerability.
3. We will provide updates on our progress and may ask for further information or assistance.
4. Once the patch is ready, we will coordinate a public disclosure and release.

We value your effort in reporting vulnerabilities and will credit you in the release notes and advisory unless you request to remain anonymous.
